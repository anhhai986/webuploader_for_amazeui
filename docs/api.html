AMUIwebuploader

用JS方式声明上传组件的方法.

var AMUIwebuploader = {
}

参数说明

template

输出上传按钮的模板

template: function (id) {
}
参数 类型 说明 备注
id 字符串 上传按钮绑定的ID名称 必填
init

初始化上传按钮

init: function (dom, obj) {
}
  • init: function (dom, obj) {
  • }
    • 参数 类型 说明 备注
    dom 字符串 手动追加上传按钮放置的位置。DOM为该标签的ID名称。 必填
    obj 对象 上传组件的对象。最少填入对象内容为：{id:'按钮的ID名称', name:'上传成功返回的隐藏域名称', pick:{id:'#按钮的ID名称 '}} 必填

    $.webuploader_for_amazeui

    本上传组件封装webuploader的核心函数，可以在别的地方调用，实现快速的上传功能。

    参数说明

    param

    核心函数必要的参数 

    $.webuploader_for_amazeui = function (param){
}
    参数 类型 说明 备注
    param 对象 上传组件的核心参数 必填

    此外param参数的最少对象传参示例如下:

    {
    id:'demo',
    name:'demo',//上传成功后文件返回相应地址的隐藏域名称。
    pick:{
        id:'#demo'
    }
}

    需要注意的是:对象参数中，id与pick中的id名称必须一致。但pick中的id前带上#号,此外name值是上传成功后文件返回相应地址的隐藏域名称，若为多个文件上传，name应该填写为数组，如：more[]
    其他的可调用参数请参考webuploader官方文档webuploader

    data-am-webuploader-simple

    简易版的上传组件

    ('[data-am-webuploader-simple]').each(function () {
}

    在任意页面的标签中，只要声明了data-am-webuploader-simple，并符合$.webuploader_for_amazeui函数的最少对象参数，则可声明上传组件。

    <div data-am-webuploader-simple="{id:'demo', name:'thumb', pick:{id:'#demo'}}"></div>

    后端开发

    上传组件对于后端提供统一的返回数据。组件附带的PHP上传库仅为演示，不能用于正式的生产环境中！返回数据格式要求如下(格式为:json)：

    {"state":"SUCCESS","msg":"\u4e0a\u4f20\u6210\u529f","original":"\u6d4b\u8bd5\u7528\u7684\u56fe\u7247.jpg","size":"1024","type":".jpg","url":"\/webuploader_for_amazeui\/server\/upload\/jhu1h21356ag.jgp"}
    参数 类型 说明 备注
    state 字符串 上传的状态。成功则请返回:'SUCCESS', 失败返回:'ERROR' 必填
    msg 字符串 提示信息，填写准确信息便于用户知道实际的问题 必填
    original 字符串 所选择上传的文件名。 非必填| 上传成功时才返回的，目前仅用于做预留扩展。
    size 字符串 所选择上传文件的大小 非必填| 上传成功时才返回的，目前仅用于做预留扩展。
    type 字符串 所选择上传文件的类型 非必填| 上传成功时才返回的，目前仅用于做预留扩展。
    url 字符串 上传成功后，后端返回上传成功的文件可访问地址。 state为SUCCESS时必填。
